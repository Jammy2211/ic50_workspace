# IC50 Workspace — Claude Instructions

This is the **IC50 cancer dose-response workspace**, the in-development
single-domain spin-out of `concr/`'s cancer pipeline. The goal is a focused,
publication-grade workflow that simulates and fits IC50 dose-response data
from the GDSC2 (Genomics of Drug Sensitivity in Cancer, v2) public release.

## Layout

```
ic50_workspace/
├── activate.sh                      # HPC venv + PYTHONPATH bootstrap
├── config/                          # placeholder for fitting configs
├── dataset/
│   ├── real/drug_<id>/              # real GDSC2 data (preprocessed per-drug)
│   └── ic50_sim/                    # simulated datasets, generated by scripts/simulator.py
├── hpc/                             # SLURM submit scripts + sync helper (see HPC section)
├── notebooks/                       # generated by PyAutoBuild from scripts/ (.py -> .ipynb)
└── scripts/
    ├── util.py                      # ALL Hill math + plotting (single source of truth)
    ├── simulator.py                 # autolens-style step-by-step simulator
    ├── ep_{sim,real}.py             # expectation-propagation fits
    ├── graphical_{sim,real}.py      # joint-search graphical-model fits
    └── profile_ep_sim.py            # category-level timing breakdown of ep_sim
```

## Profiling the EP runtime

`scripts/profile_ep_sim.py` re-runs `ep_sim.py` with monkey-patch
instrumentation that splits the wall time into Hill-fit likelihood evals,
global-fit likelihood evals, `set_model_approx` (the prior-freeze hook),
Dynesty wrapper overhead (everything inside `search.fit` that isn't a
likelihood eval), and EP-loop orchestration. Writes a markdown +  JSON
report to `scripts/results/ep_sim_profile.{md,json}` plus a scaling
projection to 100/1000/10000 datasets. Clears the cached AutoFit output
at startup so a previous `PYAUTO_TEST_MODE` run doesn't short-circuit
the proper run (see the AutoFit cache-resume note in CLAUDE memory).

## Running the simulator

From the workspace root:

```bash
python3 scripts/simulator.py
```

Produces 5 simulated datasets under `dataset/ic50_sim/dataset_{0..4}/` plus
`_sample/ground_truth.json`.

## Notebook generation

`scripts/simulator.py` is written in the heavily-docstringed autolens-workspace
style so it can be converted to a Jupyter notebook with PyAutoBuild:

```bash
PYTHONPATH=../PyAutoBuild/autobuild python3 ../PyAutoBuild/autobuild/generate.py ic50
```

Always edit `scripts/*.py`, never `notebooks/*.ipynb` directly.

## House rules

- **All Hill-curve math goes through `scripts/util.py`** (`hill_curve`,
  `log_likelihood`, `plot_dataset`). Do not re-implement the Hill function
  locally in any new script — sign convention bugs are easy to introduce
  this way.
- The simulator and any future real-data viewer must call `util.plot_dataset`
  for visualisation so simulated and real data render identically.

## Sign-convention notice (read before fitting)

`util.hill_curve` uses the **canonical decreasing** convention
`y = base / (1 + exp(n * (x - log_ic50)))`. This matches the legacy plotting
style at `concr/cancer_legacy/real/viz_hill.py` and the real-data shape (high
viability at low dose).

Concr's *production* fitting code (`concr/scripts/cancer/*.py`,
`concr/simulators/cancer_sim.py`) uses the inverted convention
`n*(log_ic50 - x)`. If concr's fitter is run on data simulated here, its
inferred `log_ic50` may need a sign-aware reinterpretation. Audit log-
likelihood signs whenever results cross the sim/concr boundary.

Known concr bugs (out of scope for this workspace, flagged for awareness):
- `concr/model_api.py:245-246` uses `+0.5 * z·z` and omits division by
  `noise_sigma` in the log-likelihood; it is not a usable likelihood and
  should not be ported into `util.py`.

## HPC runs

The `hpc/` folder mirrors `autolens_base_project/hpc/`. Submit scripts under
`batch_cpu/` and `batch_gpu/` cover the four entry points (`ep_sim`,
`ep_real`, `graphical_sim`, `graphical_real`); the `sync` helper handles
push/pull/submit/tail against the HPC.

### One-time setup

```bash
cp hpc/sync.conf.example hpc/sync.conf
# Edit hpc/sync.conf — set HPC_HOST, HPC_BASE, and (optionally) PROJECT_NAME.
# sync.conf is gitignored.
```

The remote path resolves to `$HPC_BASE/$PROJECT_NAME` (defaults to the local
folder name). Inside SLURM scripts this is the same as `$PROJECT_PATH`, so
`source $PROJECT_PATH/activate.sh` works on both ends.

### Daily workflow

```bash
# Push code (+ data on first run)
hpc/sync push

# Submit a job — pick CPU or GPU, and the entry point you want
hpc/sync submit cpu submit_graphical_sim
hpc/sync submit gpu submit_ep_real        # array job over both drugs

# Watch logs live (Ctrl+C to stop)
hpc/sync tail cpu

# After the job finishes, pull results
hpc/sync pull

# One-shot push + queue + pull-when-done
hpc/sync push-submit gpu submit_graphical_real
hpc/sync wait-and-pull 60
```

`hpc/sync` mirrors `autolens_base_project/hpc/sync` (560 LOC, copied
verbatim). See its `--help` or the autolens base-project CLAUDE.md for the
full command list (`sync`, `pull-full`, `jobs`, `sacct`, `cancel`, `du`,
`check`, `clear-logs`, etc.).

### Submit script layout

| File | Partition | Resources | Array | Calls |
|------|-----------|-----------|-------|-------|
| `batch_cpu/submit_ep_sim`         | cpu | 4 CPU / 8 GB / 4h  | —      | `scripts/ep_sim.py` |
| `batch_cpu/submit_ep_real`        | cpu | 4 CPU / 8 GB / 4h  | 0-1 (drug) | `scripts/ep_real.py --drug_id=...` |
| `batch_cpu/submit_graphical_sim`  | cpu | 4 CPU / 8 GB / 4h  | —      | `scripts/graphical_sim.py` |
| `batch_cpu/submit_graphical_real` | cpu | 4 CPU / 8 GB / 4h  | 0-1 (drug) | `scripts/graphical_real.py --drug_id=...` |
| `batch_gpu/submit_ep_sim`         | gpu | 1 GPU / 32 GB / 8h | —      | `scripts/ep_sim.py` |
| `batch_gpu/submit_ep_real`        | gpu | 1 GPU / 32 GB / 8h | 0-1 (drug) | `scripts/ep_real.py --drug_id=...` |
| `batch_gpu/submit_graphical_sim`  | gpu | 1 GPU / 32 GB / 8h | —      | `scripts/graphical_sim.py` |
| `batch_gpu/submit_graphical_real` | gpu | 1 GPU / 32 GB / 8h | 0-1 (drug) | `scripts/graphical_real.py --drug_id=...` |

CPU submits force `JAX_PLATFORM_NAME=cpu` and pin `OMP/MKL/OPENBLAS_NUM_THREADS`
to `$SLURM_CPUS_PER_TASK`. GPU submits leave JAX to pick the allocated device
and run `nvidia-smi` in the echo block.

To add more drugs to the real-data array, extend the `drugs=(...)` block and
update `--array=0-N` to match `N = len(drugs) - 1`. To use a different drug
list (e.g. a single drug for testing), shrink both accordingly.

### CPU vs GPU

For the current sample sizes (5 datasets, nlive=50, 33-param graphical fit
or 3-step EP) every script finishes in well under a minute on a laptop CPU —
the GPU partition is overkill. Use CPU for now; reach for GPU when scaling
up `nlive`, `max_steps`, or the dataset count.

### Line endings

All files under `hpc/` use **Unix LF line endings**. If you edit submit
scripts from a Windows/DOS editor, run `dos2unix` before pushing —
CRLF endings break `bash -l` headers on the HPC.
